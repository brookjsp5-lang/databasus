## 四、PITR恢复逻辑

### 4.1 PostgreSQL PITR

```
1. 解压备份文件到临时目录
2. 配置restore_command指向WAL归档目录
3. 设置recovery_target_time为目标时间
4. 创建recovery.signal文件
5. 停止PostgreSQL服务
6. 清空数据目录
7. 复制恢复文件到数据目录
8. 设置正确的文件权限
9. 启动PostgreSQL服务
10. 等待数据库就绪
```

### 4.2 MySQL PITR

```
1. 执行xtrabackup --prepare
2. 停止MySQL服务
3. 清空数据目录
4. 复制备份文件到数据目录
5. 应用BinLog到指定时间点 (mysqlbinlog)
6. 设置文件权限
7. 启动MySQL服务
8. 等待数据库就绪
```
